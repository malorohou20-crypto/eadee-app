# Prompt Claude Code — Vraies données + vrais documents pour Eadee

Copie tout ce qui suit et colle-le dans Claude Code après avoir ouvert le projet Eadee.

---

## Contexte

Aujourd'hui, l'API `api/generate-plan.js` d'Eadee génère un business plan avec Claude mais le LLM **invente parfois des chiffres marché** (hallucinations). De plus, le plan ne produit pas de **vrais documents** professionnels (CV, lettre d'intention, plan de trésorerie Excel, courrier banque). Je veux corriger ces deux problèmes.

**Stack** : HTML + Vercel Edge Functions + Supabase + Anthropic Claude API.

### Livrable 1 — Pipeline de génération en 3 étapes (search → verify → generate)

Réécris `api/generate-plan.js` en un pipeline en 3 étapes :

**Étape 1 — Recherche de données réelles** (web search + APIs)

Active l'outil `web_search` d'Anthropic pour que Claude aille chercher 3 à 5 informations clés en ligne :
- Taille du marché du secteur de l'utilisateur (en France)
- Croissance annuelle du secteur
- Concurrents principaux dans la ville visée
- Aides / subventions disponibles pour ce secteur
- Données INSEE sur la zone géographique

```javascript
import Anthropic from '@anthropic-ai/sdk';

export const config = { runtime: 'edge' };

export default async function handler(req) {
  const { idea, sector, city, budget, profile, availability } = await req.json();
  
  const client = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY });
  
  // ÉTAPE 1 — Récupérer données INSEE en parallèle
  const inseeData = await fetchINSEEData(sector, city);
  
  // ÉTAPE 2 — Demander à Claude de chercher sur le web les infos manquantes
  const searchResponse = await client.messages.create({
    model: 'claude-sonnet-4-6',
    max_tokens: 4000,
    tools: [{
      type: 'web_search_20250305',
      name: 'web_search',
      max_uses: 5
    }],
    messages: [{
      role: 'user',
      content: `Pour le projet suivant, recherche en ligne les données marché les plus récentes et fiables (sources françaises de préférence) :
      
PROJET : ${idea}
SECTEUR : ${sector}
VILLE : ${city}

Recherche en priorité :
1. Taille du marché français pour ce secteur (chiffre récent, source citée)
2. Taux de croissance annuel du secteur
3. Concurrents principaux dans ${city} ou région
4. Aides et subventions disponibles (BPI, ACRE, régionales)
5. Tendances 2025-2026 du secteur

Données INSEE pré-chargées : ${JSON.stringify(inseeData)}

Réponds en JSON structuré :
{
  "market_size": { "value": "...", "source": "...", "url": "..." },
  "growth_rate": { "value": "...", "source": "...", "url": "..." },
  "competitors": [{ "name": "...", "url": "...", "note": "..." }],
  "aids": [{ "name": "...", "amount": "...", "url": "..." }],
  "trends": ["..."],
  "verified": true
}

Si une donnée n'est pas trouvable de façon fiable, mets "value": null et "note": "Donnée non vérifiable".`
    }]
  });
  
  const verifiedData = extractJSON(searchResponse.content);
  
  // ÉTAPE 3 — Générer le plan avec les données vérifiées
  const planResponse = await client.messages.stream({
    model: 'claude-sonnet-4-6',
    max_tokens: 8000,
    system: BUSINESS_PLAN_SYSTEM_PROMPT,
    messages: [{
      role: 'user',
      content: buildPlanPrompt(idea, sector, city, budget, profile, availability, verifiedData)
    }]
  });
  
  // Stream la réponse au client
  return streamResponse(planResponse, verifiedData);
}
```

**Coût supplémentaire** : ~5 web searches × $0,01 = $0,05 par plan. Sur ton pack Empire à 19,99€ pour 8 plans, marge largement suffisante.

### Livrable 2 — Connecteur INSEE API

Crée `api/lib/insee.js` :

```javascript
// L'API INSEE est gratuite mais nécessite un token (à demander sur api.insee.fr)
// Pour démarrer simple, on utilise les data ouvertes sans auth via data.gouv.fr et stats publiques
// Si besoin d'auth, configurer INSEE_API_KEY dans Vercel

export async function fetchINSEEData(sector, city) {
  try {
    const data = {};
    
    // 1. Démographie de la ville (population, CSP)
    const cityCode = await getCityCode(city);
    if (cityCode) {
      const popResp = await fetch(`https://api.insee.fr/donnees-locales/V0.1/donnees/geo-INSEE_RP_HISTO/COM-${cityCode}`);
      // Parser et garder population, CSP+, % moins de 35 ans
      data.demographics = await popResp.json().catch(() => null);
    }
    
    // 2. Créations d'entreprises par secteur
    const apeCode = sectorToAPECode(sector);
    if (apeCode) {
      // SIRENE API : créations dernier mois par code APE
      const sirenResp = await fetch(`https://api.insee.fr/entreprises/sirene/V3/siret?q=activitePrincipaleEtablissement:${apeCode}&periode=12&nombre=1`);
      data.creations = await sirenResp.json().catch(() => null);
    }
    
    return data;
  } catch (e) {
    console.error('INSEE fetch failed', e);
    return {};
  }
}

async function getCityCode(cityName) {
  // API geo.api.gouv.fr (gratuit, sans auth)
  try {
    const r = await fetch(`https://geo.api.gouv.fr/communes?nom=${encodeURIComponent(cityName)}&limit=1`);
    const data = await r.json();
    return data[0]?.code;
  } catch { return null; }
}

function sectorToAPECode(sector) {
  const map = {
    'restauration': '5610',
    'conciergerie': '8121',
    'app': '6201',
    'saas': '6201',
    'ecommerce': '4791',
    'coaching': '8559',
    'consulting': '7022',
    'beaute': '9602',
    'sport': '9311',
    // À enrichir
  };
  const key = sector.toLowerCase();
  for (const [k, v] of Object.entries(map)) {
    if (key.includes(k)) return v;
  }
  return null;
}
```

### Livrable 5 — Génération de vrais documents annexes

Crée `api/generate-document.js` qui produit, à la demande, un vrai fichier téléchargeable :

```javascript
import Anthropic from '@anthropic-ai/sdk';

export const config = { runtime: 'nodejs' }; // pour pouvoir utiliser des libs PDF/DOCX

const TEMPLATES = {
  cv_porteur: { format: 'docx', title: 'CV Porteur de projet' },
  lettre_intention: { format: 'docx', title: 'Lettre d\'intention client' },
  courrier_banque: { format: 'docx', title: 'Courrier de présentation à la banque' },
  courrier_bpi: { format: 'docx', title: 'Demande de financement BPI' },
  plan_tresorerie: { format: 'xlsx', title: 'Plan de trésorerie 12 mois' },
  compte_resultat_3ans: { format: 'xlsx', title: 'Compte de résultat prévisionnel 3 ans' },
  fiche_synthese: { format: 'pdf', title: 'Fiche synthèse 1 page' },
};

export default async function handler(req) {
  const { documentType, planId, userId } = await req.json();
  
  // Charger le plan depuis Supabase
  const plan = await loadPlan(planId, userId);
  if (!plan) return new Response('Plan not found', { status: 404 });
  
  const template = TEMPLATES[documentType];
  if (!template) return new Response('Invalid document type', { status: 400 });
  
  // Demander à Claude de générer le contenu structuré
  const client = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY });
  const content = await generateDocumentContent(client, template, plan);
  
  // Construire le fichier réel selon le format
  let fileBuffer, contentType, extension;
  if (template.format === 'docx') {
    fileBuffer = await buildDocx(content, template.title);
    contentType = 'application/vnd.openxmlformats-officedocument.wordprocessingml.document';
    extension = 'docx';
  } else if (template.format === 'xlsx') {
    fileBuffer = await buildXlsx(content, template.title);
    contentType = 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet';
    extension = 'xlsx';
  } else if (template.format === 'pdf') {
    fileBuffer = await buildPdf(content, template.title);
    contentType = 'application/pdf';
    extension = 'pdf';
  }
  
  return new Response(fileBuffer, {
    headers: {
      'Content-Type': contentType,
      'Content-Disposition': `attachment; filename="${template.title}-${plan.name}.${extension}"`
    }
  });
}

// Implémente buildDocx avec docx (npm install docx)
// Implémente buildXlsx avec exceljs (npm install exceljs)
// Implémente buildPdf avec @react-pdf/renderer ou pdfkit
```

**Dépendances à installer** :

### Livrable 7 — Avertissements IA renforcés

Sur chaque plan généré, sur **chaque chiffre marché**, ajouter un petit indicateur visuel de fiabilité :

- 🟢 Source vérifiée (lien cliquable vers la source)
- 🟡 Estimation (à vérifier)
- 🔴 Hypothèse interne (modèle IA, à valider)

Implémentation : pendant la génération, Claude ajoute un marqueur sur chaque chiffre. Au rendu, on parse et on affiche le bon indicateur.

Exemple :
```
"Marché français : 2,4 Mds€ 🟢" → tooltip "Source : INSEE, mars 2026"
"Croissance : +31% 🟡" → tooltip "Estimation à vérifier sur le rapport sectoriel 2025"
```

Bandeau global au-dessus du plan (renforcé) :
```
⚠️ CONTENU GÉNÉRÉ PAR IA — À VALIDER AVANT TOUT DOSSIER FINANCIER
Les chiffres marqués 🟢 proviennent de sources vérifiées. Les chiffres 🟡/🔴 sont des estimations à confronter aux données officielles avant tout envoi à une banque ou un investisseur. Eadee recommande la validation par un expert-comptable.
```
